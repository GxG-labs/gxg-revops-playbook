# _staging — Knowledge Staging Area

Промежуточное хранилище для доменного знания, которое ещё не стало скиллом.

Сюда попадает всё полезное — заметки с митингов, выжимки из документов, SOP-черновики, идеи из разговоров — без давления сразу оформить в ready-to-use skill.

---

## Идея

```
[raw знание] → _staging/ → [готов к скиллу?] → {domain}/{slug}/SKILL.md
                   ↑                  ↓ нет
             _parked/         продолжаем refining
```

Staging — это **inbox**, а не архив. Не стоит хранить здесь вещи вечно. Цель — либо превратить в скилл, либо явно запарковать.

---

## Статусы заметки

| Статус | Значение |
|--------|----------|
| `raw` | Только что добавлено, не структурировано |
| `refining` | Обрабатывается — уточняется, дополняется |
| `ready` | Достаточно для scaffolding скилла |
| `parked` | Полезно, но нет ближайшего применения |

---

## Как добавить заметку

1. Скопируй `_template.md` в `_staging/{slug}.md`
2. Заполни фронтматтер (хотя бы `domain:` и `status: raw`)
3. Запиши знание в свободной форме — структура не обязательна
4. Когда захочешь продвинуть → скажи куратору: "promote {slug} to skill"

---

## Как куратор работает со staging

- `scan staging` — покажет все заметки с оценкой готовности к скиллу
- `promote {note} to skill` — scaffoldит скилл из заметки, переносит знание
- `refine {note}` — поможет структурировать сырую заметку
- `park {note}` — отметит как parked с объяснением почему

---

## Правила хранения

- Нет лимита на количество заметок в `raw` и `refining`
- `ready`-заметки старше 30 дней → куратор напоминает продвинуть
- `parked`-заметки старше 90 дней → куратор предлагает удалить или пересмотреть
- Не хранить здесь финальные скиллы — они идут в домен-папки
